TASK TYPES
VERIFY TASK
Represents a person ready for assessment.
         * Tenant

         * Guarantor

Appears ONLY when:
            * All required data for that person has been collected

            * Or sufficient alternative evidence exists (e.g. payslips/bank statements uploaded)

Never appears early.
________________


CHASE TASK
Represents a missing dependency.
Examples:
               * Tenant hasn’t completed form

               * Employer reference outstanding

               * Residential referee not responding

               * Accountant reference missing

               * Guarantor not completed

Each dependency = its own task row.
________________


TASK LIFECYCLE (HIGH LEVEL)
CREATED
  ↓
CHASE (if external dependency)
  ↓
VERIFY (when data complete)
  ↓
FINALISE → PASS | PASS WITH CONDITION | PASS WITH GUARANTOR | FAIL




At any point:
                  * Manual Send to Action Required is allowed

                  * Automatic Action Required occurs after chase exhaustion

SECTION DECISIONS (CRITICAL)
Each section allows:
                                                      * PASS

                                                      * PASS WITH CONDITION

                                                      * ACTION REQUIRED

                                                      * FAIL

ACTION REQUIRED
                                                         * Mandatory reason code

                                                         * Mandatory agent-facing note

                                                         * Optional internal note

                                                         * Correction cycle increments

                                                         * Task exits Verify immediately

PASS WITH CONDITION
                                                            * Mandatory condition text

                                                            * Appears on:

                                                               * Certificate

                                                               * Agent UI

                                                               * Audit log

                                                                  * Treated as ACCEPT/PASS operationally

FAIL
                                                                     * Requires reason

                                                                     * Hard-fail logic enforced where applicable

                                                                     * Terminates verification if terminal

________________


INCOME & AFFORDABILITY LOGIC
Always display:
                                                                        * Rent share

                                                                        * Required income (£ and ratio)

                                                                        * Confirmed income input

                                                                        * Calculated ratios:

                                                                           * Tenant: income ÷ 30

                                                                           * Guarantor: income ÷ 32

Rules:
                                                                              * Income failure alone ≠ fail

                                                                              * Outcomes:

                                                                                 * PASS

                                                                                 * PASS WITH GUARANTOR

                                                                                 * PASS WITH CONDITION (rent upfront)

                                                                                 * ACTION REQUIRED

Guarantor income applies only to their tenant’s share.
________________


CREDIT / AML / TAS (REBUILT)
TAS SCORE
                                                                                    * Starts at 999

                                                                                    * Deductions per issue

                                                                                    * No deduction for “not applicable”

HARD FAIL OVERRIDES
                                                                                       * Negative landlord/agent reference

                                                                                       * Multiple serious credit flags (e.g. CCJ + DRO)

Overrides ignore score.
SOFT DEDUCTIONS
                                                                                          * Income insufficiency

                                                                                          * Minor credit issues

                                                                                          * Behavioural inconsistencies

                                                                                          * Document quality

Small issues stack, but remain recoverable.
